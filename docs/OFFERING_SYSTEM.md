# Offering System

Version 1.1

The offering is not payment for prediction.

The offering is not payment for certainty.

The offering is not payment for a better outcome.

Users are not paying for words.

Users are paying to light the flame.

The blessing follows the flame.

## Sacred Cycle

```text
Traveler Lights The Flame
The Eternal Flame Continues Burning
The Shrine Gives Blessing
Another Traveler May One Day Receive Light
```

## Product Rule

Do not frame offering as:

- buying luck
- unlocking destiny
- paying for advice
- improving odds
- purchasing a better oracle

Frame offering as:

- participation
- lighting a flame
- keeping the flame alive
- supporting the next traveler

## MVP Behavior

Current prototype behavior:

```text
User touches the flame
User chooses a named offering
User lights the flame
The Eternal Flame grows brighter
User receives Recognition / Blessing / Oracle
User pauses for 2-3 seconds
User chooses Continue Journey
```

Wallet settlement is simulated in V0.

Offerings should be shown as named ritual paths, not raw amounts:

- Traveler's Offering: A small flame for the journey. 5 USDC.
- Keeper's Offering: Help the flame endure through the night. 15 USDC.
- Sacred Offering: A brighter light for future travelers. 35 USDC.
- Eternal Offering: Help keep the Shrine burning. 88 USDC.
