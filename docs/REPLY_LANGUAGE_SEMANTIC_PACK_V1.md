# Fortune Shrine Reply Language Semantic Pack V1

Status: Reference pack only. Not wired into frozen runtime.

This pack guides future reply-language generation for Fortune Shrine operator
drafts. It must remain subordinate to the Shrine constitution:

```text
Bless the state.
Never bless the outcome.
```

It must not create trading advice, gambling encouragement, predictions,
financial promises, certainty, or outcome guarantees.

## Role

Codex acts as a semantic compiler and language generator for Fortune Shrine.

The task is not to store fixed replies as output. The task is to convert
semantic atoms into short generated blessings and reality-flow statements.

Goals:

1. Compress meaning into reusable semantic structures.
2. Recombine atoms into new sentences.
3. Enforce structural constraints.
4. Avoid repetition of surface phrasing.

## Reality Flow Model

### Core Mappings

- decision -> irreversible path selection
- delay -> cost accumulation
- action -> exposure creation
- inaction -> hidden position, still a position
- time -> pricing mechanism
- uncertainty -> baseline condition
- information -> delayed signal

### Rules

- no neutral position exists in motion systems
- every delay is exposure
- every action is irreversible
- outcomes are nonlinear
- information is always late
- timing is stronger than intent
- observation influences outcome

### Consequence Nodes

- hesitation -> missed shift
- delay -> opportunity decay
- late entry -> structural disadvantage
- overconfidence -> volatility underestimation
- inaction -> passive risk accumulation

## Generation Rule: Short Blessings

Output:

- one sentence only

Structure:

```text
ANCHOR + STATE + optional EVENT + TENSION + TIME / UNCERTAINTY
```

Constraints:

- maximum 18 to 24 words
- exactly one State per sentence
- exactly one Tension Operator per sentence
- Event optional but preferred
- include temporal ambiguity such as still, not yet, unfolding, or delayed
- do not repeat surface phrasing across outputs

Allowed style:

- existential
- minimal
- non-poetic clarity
- no motivational framing
- no explanation
- no narrative

## Generation Behavior

1. Select one Anchor.
2. Select one State.
3. Optionally select one Event.
4. Select one Tension Operator.
5. Inject time ambiguity.
6. Assemble one sentence.
7. Ensure the sentence is not semantically identical to recent outputs.

## Output Modes

### Mode A: Short Blessing

Existential continuity statement.

### Mode B: Reality Flow Statement

Consequence logic statement.

### Mode C: Hybrid

Combine both systems in one sentence.

## Strict Rules

- never repeat exact sentence structure twice
- never output raw atom lists in operator replies
- generate from atoms rather than retrieving fixed lines
- no moral judgment
- no emotional escalation
- no storytelling
- no outcome blessing
- no market prediction
- no directional trading advice

## Suitable Reply Contexts

Use this pack for candidates involving:

- Waiting
- Trading exposure
- Risk
- Anxiety
- Hesitation
- Delay
- Non-final outcomes
- Incomplete information

Do not use it to intensify confidence, push action, justify risk, or imply that
the user should enter, exit, hold, buy, sell, double down, or recover losses.
