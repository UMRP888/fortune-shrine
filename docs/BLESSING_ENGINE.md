# Blessing Engine

Version 2.0

The Blessing Engine generates the first usable Fortune Shrine output.

It follows the canonical structure:

```text
Witness
Blessing
Echo
```

## Current Implementation

The current browser prototype uses:

- a universal Witness pool
- 70% Silent Witness lines
- 30% Explicit Witness lines
- curated universal Blessing lines
- curated universal Echo lines
- universal Blessing pools by emotional frequency
- universal Echo pools by emotional frequency
- weighted emotional frequency selection

No visible guardian, priest, oracle, avatar, guide, or character should appear in the UI.

Current output should use universal blessing and echo pools, not character-specific lines.

## Emotional Frequencies

The current weighted mix follows the Living Sky Principle:

| Frequency | Weight |
| --- | ---: |
| Warm | 40% |
| Wisdom | 20% |
| Cosmic | 15% |
| Balance | 15% |
| Melancholic | 10% |

## Output Rules

Every generated reading must:

- begin with witness
- offer blessing
- end with echo
- avoid prediction
- avoid instruction
- avoid guarantees
- avoid financial advice
- remain universal
- leave interpretation to the user

## Product Rule

The user should feel:

```text
I have been seen.
I have been blessed.
I am ready.
```

The user should not feel:

```text
I was told what to do.
I was promised success.
I received a prediction.
```
