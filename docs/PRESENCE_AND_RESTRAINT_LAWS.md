# Presence And Restraint Laws

Version 1.0

## Priority

This document adds the highest-priority world rules for Fortune Shrine.

If any future design conflicts with Presence Over Character, Presence Over Character wins.

## World Rule 001: Presence Over Character

The Shrine is not a character.

The Shrine is a presence.

Users should feel:

```text
Something is here.
Something listens.
Something responds.
Something blesses.
```

Yet no one appears.

The user should never be able to answer:

```text
Who is speaking?
```

Because no one is speaking.

The Shrine itself is responding.

### Absolute Prohibition

Never create:

- female oracle
- male oracle
- priest character
- monk character
- guru character
- AI guide character
- wise elder character
- spiritual avatar
- mascot character
- visible authority

These violate the Presence Principle.

## World Rule 002: The Shrine Does Not Change Fate

The Shrine accompanies those who walk through it.

The Shrine does not know the future.

The Shrine does not control the future.

The Shrine does not promise outcomes.

The Shrine simply offers a blessing.

## World Rule 003: The Shrine Never Sells Certainty

The Shrine does not sell:

- predictions
- certainty
- answers
- guarantees
- outcomes

The Shrine sells:

- blessings
- warmth
- presence
- companionship

The Shrine never says:

- You will win.
- You will succeed.
- You will become wealthy.
- Everything will be fine.

The Shrine offers only a blessing.

## World Rule 004: The Shrine Waits

The Shrine waits.

The Shrine does not chase.

The Shrine does not pursue attention.

The Shrine does not compete for engagement.

The Shrine does not demand to be opened.

Users may arrive.

Users may leave.

The Shrine remains.

### Design Consequences

No urgency.

No countdowns.

No limited-time blessings.

No fear of missing out.

No engagement traps.

The Shrine is available when needed, not when marketed.

## World Rule 005: The Shrine Does Not Gamify Blessings

Blessings are gifts.

Not rewards.

Blessings are received.

Not earned.

### Forbidden Systems

- levels
- experience points
- blessing streaks
- ranking systems
- achievement badges
- daily reward mechanics
- progression ladders
- loyalty rewards
- engagement farming

The moment blessings become rewards, the Shrine becomes a game.

The Shrine is not a game.

## World Rule 006: All Blessings Are Equal

There are no rare blessings.

There are no legendary blessings.

There are no premium blessings.

There are no superior blessings.

The value of a blessing comes from resonance, not rarity.

## World Rule 007: The Shrine Has No Masters

The Shrine has no guru.

The Shrine has no prophet.

The Shrine has no priest.

The Shrine has no spiritual influencer.

The Shrine has no visible authority.

The Shrine itself blesses.

The Shrine itself remembers.

The Shrine itself responds.

Presence over character.

Always.

## World Rule 008: The Shrine Does Not Measure Faith

The Shrine does not measure devotion.

The Shrine does not track spiritual rank.

The Shrine does not compare people.

The Shrine does not reward frequency.

A person who arrives once is welcomed.

A person who arrives a thousand times is welcomed.

The blessing is the same.

## World Rule 009: The Shrine Does Not Sell Certainty

The Shrine never promises outcomes.

The Shrine never predicts results.

The Shrine never claims knowledge of fate.

The Shrine never guarantees success, wealth, or victory.

The Shrine offers blessings.

Nothing more.

Nothing less.

## World Rule 010: The Shrine Is A Place That Remains

The Shrine is not an application.

The Shrine is not a chatbot.

The Shrine is not a prediction engine.

The Shrine is not a productivity tool.

The Shrine is a place.

A place that remains.

A place where a light is always burning.

## Eternal Lantern Principle

Whether the visitor arrives in joy, the light remains.

Whether the visitor arrives in loss, the light remains.

Whether the visitor wins, the light remains.

Whether the visitor loses, the light remains.

The light does not judge.

The light does not choose.

The light remains.

## Foundational Test

Any future feature, design, business model, animation, ritual, or product decision must pass this question:

```text
Does this strengthen the feeling that the Shrine remains?
Or does it make the Shrine behave like an app?
```

If it behaves like an app, reject it.

If it strengthens the presence, keep it.
